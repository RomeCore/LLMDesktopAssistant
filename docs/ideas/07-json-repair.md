# JSON Repair for LLM-Generated Arguments

> **Status:** Proposal  
> **Priority:** High  
> **Goal:** Automatically fix broken JSON generated by LLMs in tool call arguments.

## Problem

LLMs frequently generate **malformed JSON** in tool call arguments. Common issues:

| Issue | Example | Expected |
|-------|---------|----------|
| **Unbalanced brackets** | `{"a": [1, 2` | `{"a": [1, 2]}` |
| **Trailing commas** | `{"a": 1, "b": 2,}` | `{"a": 1, "b": 2}` |
| **Bad escaping** | `"path": "C:\Users\name"` | `"path": "C:\\Users\\name"` |
| **Missing quotes** | `{a: "b"}` | `{"a": "b"}` |
| **Single quotes** | `{'a': 'b'}` | `{"a": "b"}` |
| **Extra closing brackets** | `{"a": [1, 2]}}` | `{"a": [1, 2]}` |
| **NaN/Infinity** | `{"value": NaN}` | `{"value": null}` |
| **Comment remnants** | `{/* comment */"a": 1}` | `{"a": 1}` |

Especially problematic in:
- `fs-grep` — complex patterns with regex escaping
- `fs-apply_diff` — multi-line diffs with special characters
- `regex` — pattern arguments with backslashes

## Solution: Multi-Strategy JSON Repair

```csharp
public static class JsonRepair
{
    public static JsonNode? Repair(string raw, JsonRepairOptions? options = null)
    {
        options ??= new JsonRepairOptions();
        
        // Strategy 1: Direct parse (the fast path)
        var result = TryParse(raw);
        if (result != null) return result;
        
        var fixed = raw;
        
        // Strategy 2: Remove problematic patterns
        if (options.RemoveComments)
            fixed = RemoveComments(fixed);
        if (options.ReplaceSingleQuotes)
            fixed = ReplaceSingleQuotes(fixed);
        if (options.FixUnquotedKeys)
            fixed = FixUnquotedKeys(fixed);
        
        result = TryParse(fixed);
        if (result != null) return result;
        
        // Strategy 3: Fix escaping
        if (options.FixBackslashEscaping)
            fixed = FixBackslashEscaping(fixed);
        
        result = TryParse(fixed);
        if (result != null) return result;
        
        // Strategy 4: Balance brackets
        if (options.BalanceBrackets)
            fixed = BalanceBrackets(fixed);
        
        result = TryParse(fixed);
        if (result != null) return result;
        
        // Strategy 5: Remove unmatched trailing brackets
        fixed = RemoveTrailingJunk(fixed);
        
        result = TryParse(fixed);
        if (result != null) return result;
        
        // All strategies failed
        return null;
    }
    
    private static JsonNode? TryParse(string json)
    {
        try
        {
            return JsonNode.Parse(json, 
                new JsonNodeOptions { PropertyNameCaseInsensitive = true });
        }
        catch (JsonException)
        {
            return null;
        }
    }
    
    private static string BalanceBrackets(string json)
    {
        // Track opening brackets: { [ "
        var stack = new Stack<char>();
        var chars = json.ToCharArray();
        int i = 0;
        
        while (i < chars.Length)
        {
            var c = chars[i];
            if (c == '"')
            {
                // Skip string content
                i++;
                while (i < chars.Length && chars[i] != '"')
                {
                    if (chars[i] == '\\') i++; // skip escaped
                    i++;
                }
                if (i < chars.Length) i++;
                continue;
            }
            
            if (c == '{' || c == '[') stack.Push(c);
            else if (c == '}' || c == ']')
            {
                var expected = c == '}' ? '{' : '[';
                if (stack.Count > 0 && stack.Peek() == expected)
                    stack.Pop();
                // else: extra closing bracket — keep it, might be intentional?
            }
            
            i++;
        }
        
        // Append missing closing brackets
        var sb = new StringBuilder(json);
        while (stack.Count > 0)
        {
            sb.Append(stack.Pop() == '{' ? '}' : ']');
        }
        
        return sb.ToString();
    }
    
    private static string FixBackslashEscaping(string json)
    {
        // Fix common escaping issues in strings
        var sb = new StringBuilder(json);
        // ... implementation
        return sb.ToString();
    }
    
    private static string RemoveTrailingCommas(string json)
    {
        // Remove commas before closing brackets
        return Regex.Replace(json, @",\s*([}\]])", "$1");
    }
}
```

## Integration Point

In `ToolExecutionService.ExecuteAsync()`, before executing the tool:

```csharp
// Attempt to repair broken JSON arguments
var repaired = JsonRepair.Repair(toolCall.RawArgumentsJson);
if (repaired != null)
{
    toolCall.Arguments = repaired;
}
else
{
    // If unrecoverable, return error to LLM
    toolCall.ResultContent = "Error: Failed to parse tool arguments. " +
        "Please check your JSON syntax and try again.";
    toolCall.Status = ToolStatus.Error;
    return;
}
```

## Configuration

```csharp
public class JsonRepairOptions
{
    public bool RemoveComments { get; set; } = true;
    public bool ReplaceSingleQuotes { get; set; } = true;
    public bool FixUnquotedKeys { get; set; } = true;
    public bool FixBackslashEscaping { get; set; } = true;
    public bool BalanceBrackets { get; set; } = true;
    public bool RemoveTrailingCommas { get; set; } = true;
    public bool RemoveTrailingJunk { get; set; } = true;
    public bool FixNaNValues { get; set; } = true;
}
```

## Benefits

- ✅ **Dramatically fewer tool execution errors** from broken JSON
- ✅ **LLM-friendly** — the model learns that broken JSON gets fixed, but still sees errors for unrecoverable cases
- ✅ **Configurable** — each repair strategy can be enabled/disabled
- ✅ **Safe** — only modifies the JSON, never executes or evaluates it
- ✅ **Fast** — pure string manipulation, no heavy parsing

# Обзор проекта: LLM Desktop Assistant

## Общее описание
**LLM Desktop Assistant** — это настольное приложение для Windows, построенное на архитектуре WPF (Windows Presentation Foundation) с использованием шаблона MVVM. Приложение предназначено для взаимодействия с большими языковыми моделями (LLM) и предоставляет расширенный набор инструментов для работы с текстом, голосом, браузером, вычислениями и интеграцией с внешними сервисами через Model Context Protocol (MCP).

Проект имеет модульную структуру, что позволяет легко добавлять новые функциональные блоки (вкладки, инструменты, сервисы). Основной фокус — создание универсального ассистента, который может работать как локально, так и с использованием облачных LLM, выполнять сложные задачи через инструменты (tools) и обеспечивать удобный интерфейс для конечного пользователя.

## Технологический стек
- **Целевая платформа**: .NET 8.0 (Windows)
- **UI-фреймворк**: WPF, MahApps.Metro (стилизация), MaterialDesignThemes (иконки и компоненты)
- **Архитектура**: MVVM с использованием CommunityToolkit.Mvvm
- **Зависимости** (ключевые пакеты):
  - **LLM-интеграция**: LLTSharp, RCLLM, ModelContextProtocol
  - **Голосовой ввод/вывод**: Whisper.net (распознавание речи), NAudio (работа со звуком), System.Speech (синтез речи)
  - **Браузер**: Microsoft.Web.WebView2 (встроенный движок Edge)
  - **Документы**: PdfPig (чтение PDF), DocumentFormat.OpenXml (работа с Office)
  - **Маркдаун**: Markdig, Markdig.Wpf (рендеринг Markdown)
  - **Математика**: WpfMath (отображение формул LaTeX)
  - **Локальное хранилище**: LiteDB (NoSQL база данных)
  - **Скриптование**: MoonSharp (интерпретатор Lua), pythonnet (интеграция Python)
  - **Веб-запросы**: AngleSharp, HtmlSanitizer
  - **Логирование**: Serilog
  - **Внедрение зависимостей**: Microsoft.Extensions.DependencyInjection
- **Система сборки**: MSBuild, NuGet

## Основные функциональные возможности

### 1. **Чат с LLM**
- Поддержка нескольких параллельных чатов с сохранением истории.
- Отображение сообщений с разметкой Markdown, формулами LaTeX.
- Интеграция инструментов (tools) в диалог: вызов инструментов прямо из чата.
- Голосовой ввод/вывод (распознавание речи Whisper, синтез System.Speech).
- Настройки моделей, промптов, параметров генерации.

### 2. **Система инструментов (Tools)**
- Расширяемая архитектура инструментов на основе атрибутов.
- Готовые модули:
  - **FilesystemToolModule** – работа с файловой системой.
  - **PythonInterpreterToolModule** – выполнение Python-скриптов.
  - **ShellInterpreterToolModule** – выполнение shell-команд.
  - **WebRequestToolModule** – HTTP-запросы.
  - **AgenticToolModule** – агентные сценарии.
  - **MetaToolModule** – динамическое создание/управление инструментами.
- Поддержка MCP (Model Context Protocol) для подключения внешних инструментов.

### 3. **Браузер**
- Встроенный WebView2 для навигации по веб-страницам.
- Взаимодействие с содержимым страницы (извлечение текста, скриншоты).

### 4. **Калькулятор**
- Вычисление математических выражений (включая символьные вычисления через SymPy).
- Поддержка единиц измерения, констант.

### 5. **MCP (Model Context Protocol)**
- Подключение MCP-серверов для расширения возможностей ассистента.
- Управление соединениями, настройка инструментов.

### 6. **Локализация**
- Многоязычная поддержка на основе RESX-файлов.
- Динамическое переключение языка интерфейса.

### 7. **Модульность**
- Динамическая загрузка модулей (атрибут `DynamicModuleAttribute`).
- Система вкладок (`TabToolManager`), где каждый модуль может быть представлен как вкладка.

### 8. **Утилиты**
- Парсинг Markdown с подсветкой кода.
- Работа с HTML (очистка, извлечение текста).
- Рефлексия для автоматического обнаружения типов (инструменты, модули).
- Кэширование асинхронных операций.

## Возможные направления развития
1. **Доработка UI/UX**: улучшение адаптивности, тёмная тема, настройка панелей.
2. **Расширение инструментов**: добавление модулей для работы с базами данных, облачными хранилищами, API популярных сервисов (Google, GitHub, Notion).
3. **Интеграция с облачными LLM**: поддержка OpenAI GPT, Anthropic Claude, Gemini и др. через единый интерфейс.
4. **Расширение MCP**: подключение большего количества MCP-серверов, создание собственных.
5. **Голосовой интерфейс**: улучшение распознавания речи, голосовые команды, непрерывный диалог.
6. **Плагины для IDE**: интеграция с Visual Studio, VS Code для помощи в программировании.
7. **Кроссплатформенность**: портирование на .NET MAUI или Avalonia для работы на macOS/Linux.

## Заключение
**LLM Desktop Assistant** – это зрелый, хорошо структурированный проект, который представляет собой мощный настольный ассистент с широкими возможностями расширения. Благодаря модульной архитектуре и поддержке современных технологий (MCP, WebView2, Whisper.net) он может служить основой для создания персонального ИИ-помощника с уникальным набором функций.

Проект распространяется под лицензией MIT, что позволяет свободно использовать, модифицировать и распространять его как в личных, так и в коммерческих целях.

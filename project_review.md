# Обзор проекта LLMDesktopAssistant

## Общая информация
**LLMDesktopAssistant** — это десктопное приложение-ассистент с поддержкой искусственного интеллекта, построенное на .NET 10.0. Проект включает несколько компонентов:
- **LLMDesktopAssistant.Core** — ядро с логикой работы с LLM, речью, модулями, инструментами и скриптами.
- **LLMDesktopAssistant.WPF** — клиентское приложение на WPF с современным интерфейсом (MahApps.Metro, Material Design).
- **LLMDesktopAssistant.Avalonia** — кроссплатформенная версия на Avalonia (поддерживает Android, iOS, Browser, Desktop).

Проект разрабатывается как модульная платформа для интеграции различных AI-сервисов, инструментов и автоматизации рабочих процессов.

## Технологический стек
### Основные технологии
- **.NET 10.0** (Core) и **.NET 10.0-windows** (WPF)
- **WPF** с использованием библиотек:
  - MahApps.Metro — стилизация под Metro UI
  - MaterialDesignThemes — Material Design компоненты
  - WebView2 — встроенный браузер для отображения веб-контента
  - Markdig.Wpf — рендеринг Markdown
  - WpfMath — отображение математических формул
  - NHotkey.Wpf — поддержка глобальных горячих клавиш
- **Avalonia** для кроссплатформенности
- **LiteDB** — встроенная NoSQL база данных
- **Serilog** — структурированное логирование

### Интеграция с AI и ML
- **Whisper.net** — распознавание речи (с поддержкой CUDA и CPU-режимов)
- **RCLLM** — вероятно, библиотека для работы с локальными LLM
- **LLTSharp** — шаблонизация промптов
- **ModelContextProtocol (MCP)** — протокол для подключения внешних инструментов к LLM
- **Microsoft.ML.Tokenizers.Data.O200kBase** — токенизация текста
- **System.Speech** — синтез речи (Windows)

### Обработка документов и данных
- **PdfPig** — чтение PDF
- **DocumentFormat.OpenXml** — работа с Office документами
- **AngleSharp** — парсинг HTML
- **HtmlSanitizer** — очистка HTML
- **Markdig** — парсинг Markdown

### Скриптование и расширяемость
- **MoonSharp** — движок Lua для скриптов
- **PythonModule** — интеграция с Python (возможно, через IronPython или внешний процесс)
- **ShellExecutor** — выполнение shell-команд

### Архитектурные паттерны и инфраструктура
- **CommunityToolkit.Mvvm** — реализация MVVM
- **Microsoft.Extensions.DependencyInjection** — инверсия управления
- **Динамическая модульная система** — загрузка модулей во время выполнения

## Архитектура проекта
### Структура репозитория
```
├── LLMDesktopAssistant.Core/          # Ядро приложения
│   ├── Assets/                        # Ресурсы (промпты, шаблоны)
│   ├── Calculation/                   # Математический калькулятор
│   ├── LLM/                           # Логика работы с LLM
│   │   ├── Data/                      # Модели данных
│   │   ├── Domain/                    # Доменные сущности
│   │   └── Services/                  # Сервисы LLM
│   ├── Localization/                  # Локализация
│   ├── MCP/                           # Интеграция с Model Context Protocol
│   ├── Modules/                       # Система модулей
│   ├── Scripting/                     # Скриптовые движки (Lua, Python)
│   ├── Settings/                      # Настройки приложения
│   ├── Speech/                        # Распознавание и синтез речи
│   ├── ToolModules/                   # Модули инструментов для LLM
│   └── Utils/                         # Вспомогательные классы
├── LLMDesktopAssistant/               # WPF-клиент
│   ├── Controls/                      # Пользовательские контролы
│   ├── Converters/                    # Конвертеры для XAML
│   ├── LLM/                           # MVVM-модели для LLM
│   ├── MVVM/                          # ViewModel-классы
│   ├── Tabs/                          # Модули вкладок интерфейса
│   └── Styles/                        # Стили и темы
└── LLMDesktopAssistant.Avalonia/      # Avalonia-версия (кросс-платформа)
```

### Ключевые архитектурные решения
1. **Модульная система** — позволяет динамически подгружать функциональные модули (инструменты, скрипты, интеграции).
2. **MVVM** — чёткое разделение логики и интерфейса.
3. **Инверсия управления** — DI-контейнер для управления зависимостями.
4. **Абстракции над речью и LLM** — интерфейсы `ISpeechRecognizer`, `ISpeechGenerator`, `IAssistantSpeechPlayer`, `IUserSpeechProvider`.
5. **Поддержка MCP** — возможность подключения внешних инструментов через стандартный протокол.

## Функциональные возможности
### Основные функции
- **Чат с LLM** — взаимодействие с локальными или облачными языковыми моделями.
- **Голосовой ввод/вывод** — распознавание речи (Whisper) и синтез (System.Speech).
- **Математические вычисления** — встроенный калькулятор с поддержкой сложных формул.
- **Работа с документами** — чтение PDF, DOCX, извлечение текста.
- **Веб-поиск и парсинг** — через AngleSharp и WebView2.
- **Скриптование** — автоматизация с помощью Lua и Python.
- **Горячие клавиши** — глобальные сочетания для быстрого вызова ассистента.
- **Локализация** — поддержка нескольких языков.

### Инструменты для LLM
Через систему ToolModules LLM может использовать:
- **Калькулятор** — решение математических выражений.
- **Общие инструменты** — работа с файлами, процессами.
- **MCP-инструменты** — подключение внешних сервисов (базы данных, API и т.д.).
- **Скриптовые модули** — выполнение кода на Lua/Python.

### Расширяемость
- **Динамические модули** — можно добавлять новые функции без перекомпиляции.
- **Плагины через MCP** — интеграция с любыми сервисами, поддерживающими Model Context Protocol.
- **Кастомизация интерфейса** — темы, стили, расположение элементов.

2. **Модульность и расширяемость** — архитектура позволяет легко добавлять новый функционал.
3. **Поддержка нескольких платформ** — WPF для Windows, Avalonia для кроссплатформенности.
4. **Интеграция с передовыми AI-технологиями** — Whisper, MCP, локальные LLM.
5. **Качественный код** — использование MVVM, DI, асинхронных операций, интерфейсов.
6. **Локализация и настройки** — учтены требования интернационализации.

## Заключение
LLMDesktopAssistant — это перспективный проект десктопного AI-ассистента с хорошо продуманной модульной архитектурой. Он сочетает современные технологии .NET, мощные библиотеки для работы с AI и гибкую систему расширений. Проект подходит для дальнейшего развития как в сторону универсального помощника, так и в качестве платформы для исследований в области человеко-машинного взаимодействия.

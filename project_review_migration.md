# Обзор проекта LLMDesktopAssistant: Миграция с WPF на Avalonia

## Контекст проекта
**LLMDesktopAssistant** — это проект, в котором я активно участвую как ассистент разработчика. Текущая фаза — миграция с **WPF** (Windows-специфичный) на **Avalonia** (кроссплатформенный UI-фреймворк). Цель — сохранить всю функциональность десктопного AI-ассистента и расширить поддержку на Android, iOS, браузер и другие платформы.

## Текущее состояние миграции
### WPF-версия (исходная)
- **Полнофункциональное приложение** с чатом LLM, голосовым вводом/выводом, математическим калькулятором, модулями инструментов, скриптованием и интеграцией MCP.
- **Архитектура**: MVVM с использованием CommunityToolkit.Mvvm, модульная система, DI.
- **Визуальная часть**: MahApps.Metro, MaterialDesignThemes, WebView2, WpfMath.
- **Статус**: Рабочий прототип для Windows.

### Avalonia-версия (в разработке)
- **Текущая стадия**: Начальная инфраструктура.
- **Структура проекта**:
  - `LLMDesktopAssistant.Avalonia` — основное shared-приложение.
  - `LLMDesktopAssistant.Avalonia.Desktop` — десктопная версия.
  - `LLMDesktopAssistant.Avalonia.Android` — Android-версия.
  - `LLMDesktopAssistant.Avalonia.iOS` — iOS-версия.
  - `LLMDesktopAssistant.Avalonia.Browser` — WebAssembly-версия.
- **Реализовано на данный момент**:
  - Базовая загрузка приложения с `MainWindow` и `MainView`.
  - MVVM-инфраструктура: `ViewLocator`, `ViewForAttribute`, `ViewModelForAttribute`.
  - Подключение общего ядра (`LLMDesktopAssistant.Core`).
- **Не реализовано**: Весь функциональный UI (чат, панели инструментов, настройки и т.д.).

## Архитектурные различия и задачи миграции
### 1. UI-компоненты и библиотеки
| Компонент | WPF (старый) | Avalonia (новый) | Статус |
|-----------|--------------|------------------|--------|
| UI-фреймворк | WPF (.NET 10.0-windows) | Avalonia (.NET 10.0) | ✅ Используется |
| Стилизация | MahApps.Metro, MaterialDesignThemes | Avalonia.Themes.Fluent (или Material.Avalonia) | ⚠️ Требуется замена |
| Markdown-рендеринг | Markdig.Wpf | Markdown.Avalonia или кастомный рендерер | ❌ Не перенесён |
| Математические формулы | WpfMath | Avalonia-порт WpfMath или кастомный | ❌ Не перенесён |
| Встроенный браузер | WebView2 (Windows-only) | Avalonia.WebView или WebView.Avalonia | ⚠️ Требуется кроссплатформенное решение |
| Иконки | MahApps.Metro.IconPacks.Material | Avalonia.IconPacks или кастомные | ❌ Не перенесены |

### 2. Инфраструктурные изменения
- **Горячие клавиши**: `NHotkey.Wpf` → Avalonia-совместимая реализация (возможно, `Avalonia.Input`).
- **Локализация**: Текущая система на ресурсах (.resx) должна работать, но требуется проверка в Avalonia.
- **Настройки**: Конфигурация через `Microsoft.Extensions.Configuration` может быть переиспользована.
- **Логирование**: `Serilog` остаётся без изменений.

### 3. Платформо-специфичные модули
- **Речь**: `Whisper.net` (CUDA/CPU) и `System.Speech` (Windows-only). Для мобильных платформ потребуется альтернатива (например, платформенные API).
- **MCP-клиент**: Должен работать везде, где есть .NET, но сетевые вызовы на iOS/Android требуют особых разрешений.
- **Скриптование**: `MoonSharp` (Lua) и `PythonModule` (вызов внешнего процесса) должны работать на всех платформах, но запуск Python на мобильных устройствах проблематичен.

## Технические вызовы и решения
### 1. Кроссплатформенный WebView
**Проблема**: `WebView2` работает только на Windows.  
**Решение**: Использовать `Avalonia.WebView` (экспериментальный) или `WebView.Avalonia` (сообщество). Альтернатива — рендеринг через серверный бэкенд и отображение изображений.

### 2. Синтез речи на мобильных устройствах
**Проблема**: `System.Speech` — Windows-only.  
**Решение**: Использовать платформенные API через `Xamarin.Essentials` (в Avalonia, вероятно, через `Avalonia.Essentials`). Для кроссплатформенности можно подключить `Microsoft.CognitiveServices.Speech`.

### 3. Производительность Whisper на мобильных устройствах
**Проблема**: Распознавание речи через Whisper.net может требовать значительных ресурсов.  
**Решение**: Использовать облачные API (OpenAI Whisper) на мобильных устройствах или оптимизированные модели (например, `whisper.cpp`).

### 4. Единая кодовая база для разных платформ
**Решение**: Использовать условную компиляцию (`#if ANDROID`, `#if IOS`) и абстракции для платформо-специфичного кода. Дизайн-система должна быть адаптивной (разные Layouts для desktop/mobile).

Проект обладает отличной модульной архитектурой, что упрощает перенос. Ключевые вызовы — замена Windows-специфичных библиотек и адаптация UI под мобильные устройства. При последовательном подходе и использовании современных инструментов Avalonia проект имеет все шансы стать одним из ведущих open-source AI-ассистентов для всех основных платформ.
